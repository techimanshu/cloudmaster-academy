ML Concepts, Algorithms & Architectures

💡
GPT
Generative Pre-trained Transformer
TRANSFORMER
Decoder-only transformer architecture pre-trained on massive text corpora using next-token prediction (causal language modeling). Foundation of modern LLMs like ChatGPT, Claude, and Amazon Titan Text.
TypeAutoregressive decoder-only transformer
TrainingUnsupervised pre-training on next-token prediction → RLHF fine-tuning for alignment
Key IdeaSelf-attention allows each token to attend to all previous tokens; scales with data and parameters
Use Cases
Text generationSummarizationCode genQ&ATranslationChatbots
AWS LinkAmazon Titan Text (Bedrock), Claude via Bedrock, Amazon Q Developer are GPT-style models
VersionsGPT-1 (2018) → GPT-2 → GPT-3 (175B params) → GPT-3.5 → GPT-4 (multimodal)
⚡ EXAM TIPGPT = decoder-only, generative. Key mechanism: causal self-attention (each token only sees past). Contrast with BERT (encoder-only, bidirectional). Prompt engineering works because GPT continues your text.
🔬
BERT
Bidirectional Encoder Representations from Transformers
TRANSFORMER
Encoder-only transformer by Google that reads text bidirectionally (left AND right context simultaneously), pre-trained via masked language modeling (MLM) and next sentence prediction (NSP).
TypeBidirectional encoder-only transformer
Pre-trainingMasked Language Model (MLM) — predict masked tokens; Next Sentence Prediction (NSP)
Fine-tuningAdd task-specific head on top of [CLS] token representation
Use Cases
Sentiment analysisNERQuestion answeringText classificationSemantic search
VariantsRoBERTa (improved training), DistilBERT (smaller/faster), ALBERT, BioBERT (medical)
AWS LinkBERT models available in SageMaker JumpStart; used in Amazon Kendra internally
⚡ EXAM TIPBERT = understanding tasks (classification, NER, QA). GPT = generation tasks. BERT reads bidirectionally so it understands context better. Not suitable for text generation. SageMaker JumpStart hosts BERT variants.
⚙️
Transformer
Attention Is All You Need (Vaswani et al., 2017)
ARCHITECTURE
Revolutionary architecture that replaced RNNs for sequence tasks. Uses self-attention to model relationships between all tokens simultaneously — enabling massive parallelism and long-range dependencies.
Core MechanismMulti-head self-attention + feed-forward layers + positional encoding + layer norm
Attention FormulaAttention(Q,K,V) = softmax(QKᵀ / √dₖ) · V — queries, keys, values from input
Advantage over RNNParallel processing (vs sequential), no vanishing gradient, O(1) path length between any two tokens
Variants
Encoder-only (BERT)Decoder-only (GPT)Encoder-Decoder (T5, BART)
Use CasesNLP (translation, summarization, QA), vision (ViT), audio, code, multimodal
⚡ EXAM TIPTransformer = the backbone of ALL modern LLMs. Self-attention = each token looks at all others. Positional encoding = adds word order info (transformers have no inherent sequence order). Know encoder vs decoder variants.
🔁
RNN / LSTM / GRU
Recurrent Neural Networks & variants
ARCHITECTURE
Sequential neural networks that maintain a hidden state across time steps, making them naturally suited for time-series and sequential data. LSTMs and GRUs solve the vanishing gradient problem of vanilla RNNs.
RNNBasic recurrent unit; suffers from vanishing/exploding gradients for long sequences
LSTMLong Short-Term Memory — gates (input, forget, output) control what to remember/forget over long sequences
GRUGated Recurrent Unit — simplified LSTM with fewer parameters; often comparable performance
Use Cases
Time-series forecastingSpeech recognitionText generation (pre-transformer)Anomaly detectionMusic generation
LimitationSequential processing = slow training; largely superseded by Transformers for NLP tasks
AWS LinkDeepAR+ in Amazon Forecast uses LSTM; SageMaker supports RNN training
⚡ EXAM TIPRNNs process sequences step-by-step (slow). LSTMs fix vanishing gradients with memory gates. Transformers replaced RNNs for most NLP. But RNNs/LSTMs still appear in time-series (DeepAR in Forecast uses them).
🖼️
CNN / ResNet
Convolutional Neural Networks & Residual Networks
ARCHITECTURE
CNNs use convolutional filters to extract spatial hierarchies from images (edges → textures → objects). ResNet introduced skip connections (residual connections) allowing training of very deep networks (100+ layers).
CNN LayersConv → ReLU → Pooling → Flatten → FC layers. Filters learn spatial features automatically.
ResNetResidual connections: output = F(x) + x. Skip connections prevent vanishing gradients in deep networks.
Popular CNNs
ResNet-50/101VGG-16EfficientNetMobileNetInception
Use Cases
Image classificationObject detectionSegmentationMedical imagingOCR
Transfer LearningPre-trained ResNet on ImageNet → fine-tune last layers on your dataset (requires far less data)
AWS LinkSageMaker Image Classification uses ResNet; Rekognition uses CNN architectures internally
⚡ EXAM TIPCNNs = computer vision. ResNet solved "degradation problem" with skip connections. Transfer learning with pre-trained CNNs = train custom vision models with small datasets. SageMaker Image Classification algorithm = built-in ResNet.
🎭
GAN
Generative Adversarial Network
GENERATIVE
Two neural networks compete in a minimax game: a Generator creates fake data to fool the Discriminator; the Discriminator learns to distinguish real from fake. Through adversarial training both networks improve until the generator produces realistic outputs.
GeneratorTakes random noise (latent vector z) as input → outputs synthetic data (images, audio, etc.)
DiscriminatorBinary classifier: real vs fake. Provides training signal to the Generator.
Variants
DCGANStyleGANCycleGANPix2PixWGANBigGAN
Use Cases
Image synthesisData augmentationStyle transferDeepfakesDrug discoverySuper-resolution
ChallengesMode collapse, training instability, difficult hyperparameter tuning
AWS LinkStable Diffusion on Bedrock uses diffusion (related generative approach); SageMaker trains GANs
⚡ EXAM TIPGANs = adversarial generative models. Generator vs Discriminator = zero-sum game. Mode collapse = generator produces limited variety. Largely superseded by diffusion models for images. GAN training is notoriously unstable.
🌌
Diffusion Models
Denoising Diffusion Probabilistic Models (DDPM)
GENERATIVE
Generate data by learning to reverse a gradual noising process. Forward process adds Gaussian noise until data becomes pure noise; reverse process (denoising) learns to reconstruct the original. Powers Stable Diffusion and DALL-E 2.
ForwardGradually add noise over T steps: x₀ → x₁ → ... → xₜ ~ N(0,I)
ReverseNeural network (U-Net) learns to predict and remove noise: xₜ → xₜ₋₁ → ... → x₀
ConditioningText-conditioned via CLIP embeddings enables text-to-image generation
Use Cases
Image generationImage editingInpaintingVideo synthesisDrug molecule design
AWS LinkStable Diffusion available on Amazon Bedrock (Stability AI) and SageMaker JumpStart
⚡ EXAM TIPDiffusion models = current SOTA for image generation. More stable training than GANs. Stable Diffusion on Bedrock uses this architecture. Latent Diffusion Models (LDMs) operate in compressed latent space for efficiency.
🗜️
VAE
Variational Autoencoder
GENERATIVE
Generative model that learns a compressed probabilistic latent representation of data. Encoder maps input to a latent distribution; Decoder samples from it to reconstruct — enabling both compression and generation.
EncoderMaps input x to latent distribution parameters: μ (mean) and σ (std) → z ~ N(μ,σ²)
DecoderReconstructs data from sampled z; trained with reconstruction loss + KL divergence regularization
vs GANVAE = stable training, blurry outputs; GAN = unstable training, sharp outputs
Use Cases
Anomaly detectionImage generationData compressionDrug discoveryLatent space interpolation
AWS LinkVAEs used internally in Stable Diffusion's latent space; trainable on SageMaker
⚡ EXAM TIPVAE = probabilistic autoencoder. Latent space is continuous and smooth — can interpolate between data points. Great for anomaly detection (high reconstruction error = anomaly). KL divergence term regularizes latent space.
✂️
SVM
Support Vector Machine
CLASSICAL ML
Supervised learning algorithm that finds the optimal hyperplane maximizing the margin between classes. Uses the kernel trick to handle non-linearly separable data by mapping to higher-dimensional spaces.
Core IdeaMaximize margin between classes. Support vectors = data points closest to the decision boundary.
Kernel TrickRBF, polynomial, sigmoid kernels transform data to higher dimensions for non-linear classification
StrengthsEffective in high dimensions, works well with small datasets, robust to outliers
WeaknessesSlow on large datasets, kernel/C/gamma hyperparameter sensitivity, no probability output by default
Use Cases
Text classificationImage classificationBioinformaticsAnomaly detection (One-Class SVM)
AWS LinkAvailable in SageMaker built-in algorithms; scikit-learn SVM trainable on SageMaker
⚡ EXAM TIPSVM = classical ML for classification/regression. C parameter controls bias-variance tradeoff (larger C = less regularization). One-Class SVM = novelty/anomaly detection. Mostly replaced by tree ensembles and DL for tabular data.
🏆
XGBoost
Extreme Gradient Boosting
CLASSICAL ML
Highly optimized gradient boosting library that sequentially builds an ensemble of decision trees, each correcting errors of the previous. Dominant algorithm for structured/tabular data in ML competitions and industry.
How It WorksAdditive training: each new tree fits the residual errors (negative gradients) of the ensemble so far
Key Params
n_estimatorsmax_depthlearning_rate (eta)subsamplecolsample_bytreelambda/alpha (L2/L1)
StrengthsHandles missing values natively, built-in regularization, fast (parallel tree construction), feature importance
Use Cases

Bias–Variance Tradeoff
Overfitting, Underfitting & Regularization
FUNDAMENTALS
Core ML tradeoff: high bias = underfitting (model too simple, misses patterns); high variance = overfitting (model memorizes training data, fails on new data). Goal: find the sweet spot.
UnderfittingHigh training error + high validation error. Fix: more complex model, more features, less regularization.
OverfittingLow training error + high validation error. Fix: more data, dropout, L1/L2 regularization, early stopping, cross-validation.
Regularization
L1 (Lasso) — sparsityL2 (Ridge) — weight decayDropoutEarly stoppingData augmentation
Cross-Validationk-fold CV: split data into k folds, train on k-1, test on 1, rotate — more reliable evaluation than single split
AWS LinkSageMaker Automatic Model Tuning (HPO) searches for optimal regularization hyperparameters
⚡ EXAM TIPMassive exam topic. Train accuracy high, test accuracy low → overfitting. Both low → underfitting. Dropout = randomly zero out neurons during training (regularization). L1 = feature selection. L2 = shrinks weights. SageMaker HPO automates hyperparameter search.

More GenAI Concepts
Tokens, Context Window, Temperature, Top-P, Hallucination
BEDROCK
Core vocabulary every AI Practitioner must know — how LLMs represent and process text, key inference parameters that control output, and the fundamental limitations to be aware of.
TokenSmallest unit of text an LLM processes (~4 chars / 0.75 words). Input + output tokens = total cost.
Context WindowMax tokens a model can process at once (input + output). E.g., Claude 3 = 200K tokens. Older models = 4K–32K.
Temperature0 = deterministic/factual. 1 = random/creative. Range 0–1 (sometimes 0–2). Controls sampling randomness.
Top-P (nucleus)Sample from the smallest set of tokens whose cumulative probability ≥ P. Top-P=0.9 = more focused than 1.0.
Top-KSample from the K most likely next tokens only. Lower K = more focused output.
HallucinationLLM generates confident but factually incorrect information. Mitigate with RAG, Guardrails grounding check, lower temperature.
Inference Params
max_tokenstemperaturetop_ptop_kstop_sequences
⚡ EXAM TIPTemperature=0 → always most likely token (good for factual QA/code). Temperature=1 → creative writing. Top-P and Top-K both filter the sampling distribution. Hallucination is the #1 GenAI risk — answer: RAG + Guardrails. Context window limits = why chunking in RAG matters.

Phases of a Machine Learning Project
End-to-end ML lifecycle
ML PROCESS
A structured ML project follows well-defined phases from business understanding through deployment and monitoring. Skipping phases is the #1 cause of ML project failure.
1. Business UnderstandingDefine the problem, success metrics, ROI, and whether ML is appropriate. Identify stakeholders.
2. Data CollectionGather raw data from databases, APIs, sensors, third-party providers. Assess data quality and quantity.
3. Data PreparationClean (handle missing values, outliers), transform (normalize, encode categoricals), feature engineer, split train/val/test.
4. Model TrainingSelect algorithm, set hyperparameters, train on training set, evaluate on validation set, iterate.
5. Model EvaluationAssess on held-out test set using business-relevant metrics. Check for bias with Clarify.
6. DeploymentPackage model, deploy to endpoint (real-time, serverless, async, or batch). A/B test with shadow deployment.
7. MonitoringTrack data drift, model quality degradation, bias drift with SageMaker Model Monitor. Retrain as needed.
AWS MappingGround Truth → Glue/DataBrew → SageMaker Training → Clarify → Model Registry → Endpoint → Model Monitor
⚡ EXAM TIPKnow all 7 phases and the AWS service for each. Data prep is typically 80% of project time. Test set must NEVER be seen during training. Model Monitor catches degradation post-deployment. The loop is iterative — models need retraining when data distribution shifts.

Hyperparameters
Configuration values set before training begins
ML PROCESS
Hyperparameters are the "knobs" you configure before training — unlike model parameters (weights) which are learned. Finding the best hyperparameters is called hyperparameter optimization (HPO) or tuning.
Common HPs
Learning rateBatch sizeEpochsHidden layers/unitsDropout rateL1/L2 lambdaMax depth (trees)
Learning RateControls step size in gradient descent. Too high → divergence. Too low → slow convergence. Most critical HP.
Batch SizeSamples per gradient update. Large batch = stable gradients, needs more memory. Small = noisy but often generalizes better.
EpochsFull passes over training data. Too many → overfitting. Use early stopping to find optimal epoch automatically.
HPO MethodsGrid search (exhaustive), Random search (efficient), Bayesian optimization (smart, learns from trials)
AWS HPOSageMaker Automatic Model Tuning (AMT) uses Bayesian optimization to find best hyperparameters efficiently
⚡ EXAM TIPHyperparameters = set BEFORE training (not learned). Parameters = weights learned DURING training. SageMaker Automatic Model Tuning = Bayesian HPO. If model overfits → increase regularization HPs (dropout, L2 lambda, reduce epochs). If underfits → more capacity (larger network, more epochs).

Deployment modes: Real-time, Serverless, Async, Batch

Inference is using a trained model to make predictions on new data. SageMaker offers four deployment modes to match different latency, scale, and cost requirements.

When is ML NOT Appropriate?
Knowing when NOT to use machine learning
ML PROCESS
ML is powerful but not always the right tool. Understanding when traditional software, rules-based systems, or simpler analytics are better is a key practitioner skill.
Insufficient DataML needs enough labeled examples to generalize. Rule of thumb: fewer than hundreds of examples → use rules or heuristics instead.
Explainability RequiredRegulated industries (healthcare, finance, legal) may require fully explainable decisions → simple rule-based systems or linear models.
Problem is DeterministicIf the problem has a perfect algorithmic solution (sorting, routing, arithmetic) — use the algorithm. ML adds unnecessary complexity.
Cost > BenefitIf ML infrastructure, data labeling, and maintenance cost exceeds the business value delivered — don't use ML.
Static/Small ProblemIf rules rarely change and there are <20 if-else rules that cover all cases — maintain the rules.
Real-time Hard RulesSafety-critical systems where probabilistic errors are unacceptable (aircraft control, drug dispensing). Combine ML with hard guardrails.
⚡ EXAM TIPClassic exam scenario: "Company wants to use ML for X — is this appropriate?" Red flags: no training data, fully deterministic problem, need 100% explainability, marginal business value. When in doubt: is there a pattern in data that ML can learn? If yes → ML. If it's just rules → code the rules.

Model Evaluation Metrics — Deep Dive
Confusion matrix, ROC-AUC, BLEU, ROUGE, METEOR, BERTScore
ML PROCESS
Comprehensive reference for all evaluation metrics across ML task types — classification, regression, NLP generation, and retrieval. Choosing the wrong metric can lead to deploying bad models.
Confusion MatrixTP (correct positive), TN (correct negative), FP (false alarm / Type I), FN (missed detection / Type II)
PrecisionTP / (TP+FP) — "Of all predicted positives, how many were correct?" Minimize FP. Use when false alarms are costly.
Recall (Sensitivity)TP / (TP+FN) — "Of all actual positives, how many did we catch?" Minimize FN. Use for fraud/disease detection.
F1 Score2·P·R / (P+R) — harmonic mean of precision and recall. Use when class imbalance exists.
AUC-ROCArea under ROC curve (TPR vs FPR). 0.5 = random, 1.0 = perfect. Threshold-independent classification metric.
BLEUPrecision-based n-gram overlap (translation). Compares generated text to reference. Range 0–1.
ROUGERecall-based n-gram overlap (summarization). ROUGE-1 (unigrams), ROUGE-2 (bigrams), ROUGE-L (longest common subseq).
BERTScoreUses BERT embeddings to compute semantic similarity between generated and reference text. Better than n-gram metrics.
PerplexityHow well the model predicts a sample. Lower = better. Measures language model quality. Not used for task-specific evaluation.
⚡ EXAM TIPScenario: "minimize missed fraudulent transactions" → maximize Recall. "minimize false fraud alerts to customers" → maximize Precision. Imbalanced dataset → don't use Accuracy. BLEU = translation. ROUGE = summarization. BERTScore = semantic quality (best for GenAI). Bedrock Evaluation uses ROUGE + BERTScore.

ML Terms You May Encounter
Glossary of key ML terminology for the exam
ML PROCESS
Essential ML vocabulary tested on the AI Practitioner exam — from foundational math concepts to practical ML operations terms.
SupervisedTraining on labeled data (input→output pairs). Examples: classification, regression.
UnsupervisedFind patterns in unlabeled data. Examples: clustering (K-Means), dimensionality reduction (PCA), anomaly detection.
Semi-SupervisedSmall labeled dataset + large unlabeled dataset. Common when labeling is expensive.
Self-SupervisedLabels generated from the data itself. BERT (masked tokens), GPT (next token prediction). Foundation of LLMs.
Reinforcement LearningAgent learns by trial-and-error maximizing reward signal. RLHF uses RL to align LLMs to human preferences.
Transfer LearningTake a pre-trained model and fine-tune on new task. Requires far less data and compute than training from scratch.
Gradient DescentOptimization algorithm that iteratively updates weights in direction of negative gradient (steepest descent) to minimize loss.
BackpropagationAlgorithm to compute gradients through the network using chain rule — enables gradient descent in deep networks.
NormalizationScale features to same range (Min-Max: 0–1) or distribution (Z-score: μ=0, σ=1). Critical for distance-based models.
Dimensionality ReductionPCA, t-SNE, UMAP — reduce feature count while preserving information. Helps visualization and reduces overfitting.
⚡ EXAM TIPKnow the 4 learning paradigms: Supervised (labeled), Unsupervised (unlabeled), Semi-supervised (few labeled), Reinforcement (reward). Self-supervised = how LLMs are pre-trained. Transfer learning = why fine-tuning works. Normalization is a data preprocessing step, not model training.

GenAI Challenges
Risks, limitations & responsible deployment concerns
RESPONSIBLE AI
Generative AI introduces unique challenges beyond traditional ML — understanding these risks is essential for the AI Practitioner exam and for responsible real-world deployment.

Prompt InjectionMalicious user input overrides system prompt instructions. Mitigation: Bedrock Guardrails prompt attack detection, input validation.

Cost UnpredictabilityToken costs can spike with complex prompts. Mitigation: set max_tokens, use Batch Inference, monitor with CloudWatch.
⚡ EXAM TIPHallucination = biggest GenAI risk. Fix = RAG + Guardrails. Prompt injection = security attack on LLMs. Fix = Guardrails prompt attack detection. Data privacy = PII in prompts/outputs. Fix = Comprehend PII redaction + Guardrails sensitive info. Know the risk → mitigation pairs.

S3 EndpointGateway endpoint for S3 — free, routes S3 traffic through AWS network. Critical for training jobs in private subnets that need S3 data.
Security GroupsControl inbound/outbound traffic for SageMaker notebook instances, training jobs, and endpoints. Default: deny all.
NACLsSubnet-level stateless firewall — additional layer of network control for ML subnets.
Bedrock NetworkBedrock API calls can be made via VPC interface endpoint (com.amazonaws.region.bedrock-runtime) to avoid public internet.
Data EncryptionIn-transit: TLS 1.2+ for all AWS API calls. At-rest: KMS encryption for S3, EBS volumes (notebooks), SageMaker models.
⚡ EXAM TIPVPC + no internet access + S3 Gateway Endpoint = most secure SageMaker training setup. Without S3 Gateway Endpoint, private subnet training jobs CANNOT access S3 training data. Bedrock VPC endpoint = com.amazonaws.REGION.bedrock-runtime. Always use KMS for data encryption in regulated industries.

Responsible AI — AWS Framework
AWS's 8 dimensions of responsible AI
RESPONSIBLE AI
AWS defines responsible AI across eight core dimensions. Understanding these pillars and the AWS services that address each is essential for the AI Practitioner exam.
FairnessEnsure model doesn't discriminate based on protected attributes. → SageMaker Clarify (bias detection)
ExplainabilityUnderstand why the model made a prediction. → Clarify SHAP values, SageMaker Model Cards
Privacy & SecurityProtect user data and model artifacts. → KMS, VPC, IAM, Guardrails sensitive info filtering, Comprehend PII
SafetyPrevent harmful outputs and misuse. → Bedrock Guardrails, content filtering, human-in-the-loop with A2I
ControllabilityHumans can oversee and intervene in AI decisions. → SageMaker Model Monitor, A2I (Augmented AI), approval workflows
Veracity & RobustnessModel performs reliably across conditions. → Model Monitor (drift), adversarial testing, SageMaker Clarify
TransparencyDisclose AI involvement and model limitations. → SageMaker Model Cards, watermarking (Titan), documentation
GovernanceOrganizational policies and oversight. → Model Registry approval workflows, Audit Manager, CloudTrail, SageMaker Role Manager
⚡ EXAM TIPKnow all 8 pillars and their AWS service mappings. Fairness + Explainability = SageMaker Clarify. Safety = Bedrock Guardrails. Controllability = human-in-the-loop (A2I). Transparency = Model Cards + watermarking. Governance = Model Registry + Audit Manager. Heavily tested!

Amazon's Custom AI Hardware
Trainium, Inferentia, and GPU instances
HARDWARE
AWS offers purpose-built AI chips and GPU instances at every price/performance point — from cost-optimized custom silicon to the highest-performance GPU clusters for frontier model training.
AWS Trainium (Trn1)AWS-designed chip optimized for training deep learning models. Up to 50% cost savings vs GPU alternatives. Best for training LLMs and large CNNs. Uses Neuron SDK.
AWS Inferentia (Inf1/Inf2)AWS-designed chip optimized for inference (not training). Lowest cost per inference on AWS. Inf2 supports LLM inference. Neuron SDK required.
P-series (GPU)p4d.24xlarge (A100 GPUs), p3.16xlarge (V100) — NVIDIA GPUs for training. Flexible but expensive.
G-series (GPU)g4dn (T4 GPUs), g5 (A10G) — for graphics, inference, and lighter training workloads.
SageMaker HyperPodManaged cluster for large-scale distributed training. Automatically recovers from node failures during multi-week training runs.
EC2 UltraClustersThousands of Trn1/P4d instances connected via 400 Gbps EFA networking for petabyte-scale model training
Neuron SDKAWS SDK to compile and optimize PyTorch/TensorFlow models for Trainium and Inferentia chips
⚡ EXAM TIPTrainium = TRAINING custom models cheaply. Inferentia = INFERENCE (running models) cheaply. Both use Neuron SDK. GPU (P/G instances) = flexible, more expensive. SageMaker HyperPod = fault-tolerant distributed training cluster. Key: Trainium for training, Inferentia for inference = cost optimization.
