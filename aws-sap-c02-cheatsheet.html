Exam tips & traps
⚡ TRAP
Spot vs Savings Plans: Spot for fault-tolerant / batch / flexible-start jobs. Savings Plans for steady-state, always-on workloads. Questions disguise cost-cutting as "design" problems — look for words like "fluctuating" or "interruption-tolerant".
⚡ TRAP
CloudWatch vs CloudTrail: CloudWatch = metrics, logs, alarms (operational visibility). CloudTrail = API call history (auditing / compliance). If question is about "who did what" → CloudTrail. If "is my service healthy" → CloudWatch.
⚡ TRAP
DAX vs ElastiCache: DAX is DynamoDB-specific, write-through, microsecond latency. ElastiCache is general-purpose (Redis/Memcached). If question mentions DynamoDB read latency → DAX first.
⚡ TRAP
Reserved Instance flexibility: Convertible RIs can change instance family/OS/tenancy. Standard RIs cannot. Compute Savings Plans are even more flexible than Convertible RIs — no instance family lock-in.
⚡ TRAP
Global Accelerator vs CloudFront: GA = static IPs, TCP/UDP, non-HTTP, gaming, VoIP. CloudFront = HTTP/HTTPS caching, Lambda@Edge. Both improve global performance but for different use cases.

The 7 Rs of migration
Retire
Decommission unused / low-value apps. Often overlooked — up to 10–20% of portfolio.
Retain
Keep on-prem (compliance, latency, recent investment). Not a failure — valid strategy.
Rehost
Lift & shift to EC2. Fast, no code change. Use MGN (Application Migration Service).
Replatform
Lift, tinker & shift. e.g. move to RDS instead of self-managed DB. Minimal changes.
Repurchase
Move to SaaS (Salesforce, Workday). For commodity apps where custom code has no value.
Refactor
Re-architect for cloud-native. Highest effort + benefit. Triggers: scale, agility, features.
Relocate
Move to VMware Cloud on AWS or similar. Minimal changes to existing VM-based apps.
Exam tips & traps
⚡ TRAP
DMS + SCT: Heterogeneous migrations (Oracle → Aurora PostgreSQL) need SCT first for schema conversion, then DMS for data. Homogeneous (MySQL → Aurora MySQL) → DMS only.
⚡ TRAP
ECS vs EKS: ECS = simpler, AWS-native, no Kubernetes knowledge needed. EKS = Kubernetes compatibility, multi-cloud portability. Choose EKS when question mentions "existing Kubernetes" or "portable".
⚡ TRAP
SQS vs SNS vs EventBridge: SQS = pull/queue (decoupling, at-least-once). SNS = push/fan-out to multiple endpoints. EventBridge = event routing with rules, schema registry, cross-account/SaaS events.
⚡ TRAP
Step Functions vs SQS: Step Functions = workflow orchestration with state, retries, error handling. SQS = simple async decoupling. If the question shows a multi-step process with branching → Step Functions.

Exam tips & traps
⚡ TRAP
VPC Peering vs Transit Gateway: Peering = 1:1, no transitive routing, scales poorly. TGW = hub-and-spoke, transitive routing, scales to thousands of VPCs. Choose TGW when ≥3 VPCs need to communicate or you need centralized routing.
⚡ TRAP
CloudHSM vs KMS: KMS = AWS-managed HSM (shared), FIPS 140-2 Level 2. CloudHSM = dedicated hardware, you control keys, FIPS 140-2 Level 3. Required when question mentions "you must manage your own keys" or "FIPS Level 3".
⚡ TRAP
RDS Multi-AZ vs Read Replica: Multi-AZ = synchronous standby for failover (HA). Read Replica = asynchronous copy for read scaling. Can promote Read Replica to standalone DB. Multi-AZ does NOT improve read performance.
⚡ TRAP
ALB vs NLB vs GLB: ALB = HTTP/HTTPS, path/host-based routing, WebSockets. NLB = TCP/UDP, static IPs, preserve source IP, lowest latency. GLB = bumps traffic through third-party network appliances (firewalls, IDS).
⚡ TRAP
Kinesis Data Streams vs Firehose: KDS = real-time, you manage consumers, 1–365 day retention. Firehose = near-real-time (60s buffer), serverless delivery to S3/Redshift/OpenSearch. Firehose is simpler but adds latency.
⚡ TRAP
S3 storage classes: Standard → Standard-IA → One Zone-IA → Intelligent-Tiering → Glacier Instant → Glacier Flexible Retrieval → Glacier Deep Archive. One Zone-IA loses data if AZ fails — only for reproducible data.

Exam tips & traps
⚡ TRAP
SCPs do not grant permissions. SCPs only restrict what member accounts CAN do. An SCP Allow just removes an SCP deny. Without IAM policies, even SCP Allow = no access. Management account is NEVER affected by SCPs.
⚡ TRAP
RAM vs VPC Peering: RAM shares actual resources (subnets, TGW, Route 53 Resolver rules). VPC Peering shares network connectivity. Share subnets via RAM so teams deploy into a shared networking account — this is the recommended multi-account network pattern.
⚡ TRAP
AD Connector vs Managed AD: AD Connector = proxy, no user data stored in AWS, on-prem AD must be reachable. Managed AD = full AD in AWS, works if on-prem goes down. If question says "no replication to cloud" or "on-prem AD stays authoritative" → AD Connector.
⚡ TRAP
CloudFormation StackSets + Organizations: SELF_MANAGED mode needs you to set up trust manually. SERVICE_MANAGED mode uses trusted access with Organizations — automatically deploys to new accounts added to target OUs.
⚡ TRAP
IAM Identity Center vs IAM SAML federation: Identity Center = modern, preferred for workforce SSO, stores permission sets, works across all accounts. SAML federation = legacy, per-account, you manage everything. Always prefer Identity Center in new designs.

Exam tips & traps
⚡ TRAP
MGN vs DMS: MGN = server/VM migration (any OS, lift & shift). DMS = database migration (with optional schema conversion). They're complementary — MGN for the server, DMS for the database inside it. SMS is deprecated; always choose MGN.
⚡ TRAP
DataSync vs Storage Gateway: DataSync = one-time or scheduled bulk transfer (set it, run it, done). Storage Gateway = permanent hybrid access, local cache, ongoing integration. If question says "ongoing" or "hybrid access" → Storage Gateway. "Migrate data" → DataSync.
⚡ TRAP
Snowball sizing math: Snowball Edge = 80TB usable per device. If question says 200TB → 3 Snowballs. Snowmobile for >10PB. Always calculate: at 100 Mbps, 80TB takes ~74 days — always use offline for large datasets over slow links.
⚡ TRAP
ADS agent vs agentless: Agent = installed on each server, detailed data (processes, network connections, CPU/RAM over time). Agentless = VMware vCenter only, less detail, no process-level data. Agent is preferred for accurate dependency mapping.
⚡ TRAP
Migration Evaluator vs Pricing Calculator: Migration Evaluator = business case, TCO analysis, uses real utilization data. Pricing Calculator = estimate AWS costs for a given architecture. Use Evaluator for "should we migrate" decisions, Pricing Calculator for "how much will this design cost".
